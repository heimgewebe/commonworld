# Static Catalog API Route Fixtures

## Status

- Task: COMMONWORLD-ATLAS-V1-T013
- Type: static route fixture, no runtime implementation
- Builds on: `docs/blueprints/catalog-api-contract.md` and `examples/commonworld/catalog-export.sample.json`
- Fixture anchor: `examples/commonworld/catalog-api-route-fixtures.sample.json`
- Boundary: no API server, no database, no ingestion worker, no public submissions, no write path

## Decision

commonworld may generate static route fixtures for the read-only catalog API contract before it builds any API runtime.

The route fixture is a deterministic sample of what the future read-only API routes may return. It is not a server, router, handler, deployment target, persistence layer or publication workflow.

```text
catalog API contract       = allowed future read-only delivery shape
static route fixture       = generated example responses for those routes
catalog API implementation = not introduced by this slice
```

## Covered read-only routes

The static fixture must cover exactly these contract routes:

- `GET /catalog/v1/catalog-export` as a static response over the generated catalog export;
- `GET /catalog/v1/projects` as a static project summary list derived from the generated catalog export;
- `GET /catalog/v1/projects/{project_id}` as one static project detail fixture for a known CommonProject identifier.

The detail fixture uses a concrete request path such as `/catalog/v1/projects/neighborhood-repair-circle-fixture` while preserving the route template `/catalog/v1/projects/{project_id}`.

## Required boundary

Every route fixture must declare:

- `method: GET`;
- `access: public-read-only`;
- `writes: false`;
- `submissions: false`;
- `auth_required: false`;
- `status: 200` for the covered successful static examples;
- a response body derived from `examples/commonworld/catalog-export.sample.json` or an accepted CommonProject file.

## Prohibited shortcuts

T013 must not introduce:

- POST, PUT, PATCH or DELETE routes;
- API server, router, handler or middleware;
- database requirement;
- ingestion worker;
- public submission route;
- account, session or role system;
- review write path;
- import mutation path;
- weltgewebe write path.

## Generation rule

`examples/commonworld/catalog-api-route-fixtures.sample.json` is generated by `scripts/generate_catalog_api_route_fixtures.py`.

A stale fixture means the static sample no longer matches the current catalog export or route contract. Validation must fail until the fixture is regenerated.
